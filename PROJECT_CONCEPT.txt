1. TUJUAN UTAMA
Package UI kustom yang fleksibel, elegan, modern, dan mandiri.
Install via GitHub private:
  flutter_ui_kit:
    git:
      url: git@github.com:USERNAME/flutter_ui_kit.git
      ref: v1.0.0

2. KONSEP INTI (CORE CONCEPTS)
- Responsivitas Tinggi: AppScale (sp, w, h, r); desktop ≥1024px tanpa scaling
- Sistem Desain: AppColors light/dark; toggle via ViewDemoUiController (demo)
- Clean & Modern UI: Glassmorphism, animasi halus, sudut membulat
- GetX wajib: AppScale (Get.width), AppColors (Get.isDarkMode), AppDialog, AppSnackbar

3. ATURAN PENTING KODE (RULES)
- "Jika Null, Maka Hidden": parameter null tidak dirender + spacing bersyarat
- Reactive State: Obx/GetX di demo; controller .obs / Rxn<T>
- isLoading:
  * Cards & data display → Skeletonizer / Skeleton.replace / Bone
  * Input interaktif → CircularProgressIndicator atau disable, BUKAN Skeletonizer

9. LANGKAH PUSH KE GITHUB (PRIVATE REPO)
----------------------------------------------------
Prasyarat:
- Akun GitHub + repo private kosong (mis. nama: flutter_ui_kit)
- Git terinstall di mesin lokal
- Autentikasi: SSH key ATAU Personal Access Token (PAT) dengan scope repo
- Flutter SDK terinstall (untuk verifikasi sebelum push)

A. Persiapan folder sebelum commit
----------------------------------
1. Pastikan tidak ada proses flutter/dart yang masih berjalan
2. Bersihkan artefak build:
     cd <folder_proyek>
     flutter clean
     cd example && flutter clean && cd ..
3. (Opsional) Verifikasi terakhir:
     flutter pub get
     flutter analyze
     flutter test
     cd example && flutter pub get && flutter analyze && flutter test

B. Buat repository private di GitHub
--------------------------------------
1. Login GitHub → New repository
2. Nama: flutter_ui_kit (atau sesuai keinginan)
3. Visibility: Private
4. JANGAN centang "Add README" / .gitignore (sudah ada lokal)
5. Salin URL repo:
   - SSH:  git@github.com:USERNAME/flutter_ui_kit.git
   - HTTPS: https://github.com/USERNAME/flutter_ui_kit.git

C. Inisialisasi Git & commit pertama (jika belum ada git)
---------------------------------------------------------
cd <folder_proyek>

git init
git branch -M main

# Cek file yang akan di-commit (pastikan .dart_tool/ & build/ tidak ikut)
git status

git add .
git commit -m "Initial release: flutter_ui_kit v1.0.0"    

D. Hubungkan remote & push
--------------------------
# SSH (disarankan)
git remote add origin git@github.com:USERNAME/flutter_ui_kit.git

# ATAU HTTPS
# git remote add origin https://github.com/USERNAME/flutter_ui_kit.git

git push -u origin main

E. Tag versi (untuk pinning di consumer pubspec)
------------------------------------------------
git tag v1.0.0
git push origin v1.0.0

# Tag berikutnya saat rilis baru: v1.0.1, v1.1.0, dst.

F. Pasang di project Flutter lain (consumer)
--------------------------------------------
pubspec.yaml consumer:

dependencies:
  flutter:
    sdk: flutter
  flutter_ui_kit:
    git:
      url: git@github.com:USERNAME/flutter_ui_kit.git
      ref: v1.0.0        # gunakan tag, atau branch: main
  get: ^4.7.3
  flutter_screenutil: ^5.9.3

Lalu:
  flutter pub get

Setup main.dart consumer (WAJIB):
  ScreenUtilInit → GetMaterialApp → AppTheme.lightTheme / darkTheme

Import:
  import 'package:flutter_ui_kit/flutter_ui_kit.dart';

Catatan akses private repo:
- SSH: pastikan ssh key terdaftar di GitHub account
- HTTPS: git akan minta username + PAT (bukan password akun)
- CI/CD: simpan PAT atau deploy key di secrets

G. Update rilis berikutnya (workflow rutin)
--------------------------------------------
1. Ubah version di pubspec.yaml (mis. 1.0.0+1 → 1.0.1+1)
2. Update CHANGELOG.md
3. flutter analyze && flutter test (root + example)
4. git add .
5. git commit -m "Release v1.0.1: <ringkasan perubahan>"
6. git push origin main
7. git tag v1.0.1 && git push origin v1.0.1
8. Consumer update ref di pubspec: ref: v1.0.1 → flutter pub get

I. Troubleshooting singkat
---------------------------
- "Permission denied (publickey)" → setup SSH key di GitHub
- "Repository not found" → cek nama repo + akses private + akun yang login
- Consumer gagal pub get → pastikan PAT/SSH punya akses read ke repo private
- Setelah clone/zip: jalankan flutter pub get di root DAN di example/
